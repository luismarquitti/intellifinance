**Agent**: Writer

**Gate**: User approval required before PR creation

## ğŸš€ Getting Started

### Prepare Input Materials

```powershell
# Create input folder for your JIRA issue
New-Item -ItemType Directory -Path ".analysis-inputs/HPXAPPS-12345" -Force

# Add logs, screenshots, notes
Copy-Item "error.log" ".analysis-inputs/HPXAPPS-12345/logs/"
Copy-Item "screenshot.png" ".analysis-inputs/HPXAPPS-12345/screenshots/"

# Add user notes
@"
## Investigation Notes
- Issue started after recent deployment
- Only affects premium users
- Peak hours: 2-4pm EST
"@ | Out-File ".analysis-inputs/HPXAPPS-12345/notes.md"
```

### Start the Dashboard (Live Reload)

```powershell
cd analysis-workspace
npm start
```

The dashboard will open at `http://localhost:3001` with **hot-reloading** enabled. Any changes agents make to files in `docs/` will automatically update the dashboard in real-time.

**Alternative Port**: If you need a different port, use:
```powershell
npm run start:custom -- --port 3005  # Use any port you prefer
```

### Analyze the Issue

Ask HP Dev Agent (Orchestrator):

```
"@HP Dev Agent, analyze JIRA issue HPXAPPS-12345 using materials in .analysis-inputs/HPXAPPS-12345/"
```

The agent will:
1. Read inputs from `.analysis-inputs/HPXAPPS-12345/`
2. Create output folder `docs/HPXAPPS-12345/`
3. Generate analysis report with references to input sources
4. Update dashboard automatically (hot-reload)
5. Display analysis in sidebar
