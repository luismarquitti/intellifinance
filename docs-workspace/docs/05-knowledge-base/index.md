## Contributing

To add content to the Knowledge Base:

1. Create a markdown file in the appropriate subdirectory
2. Include proper YAML frontmatter
3. Follow Docusaurus formatting standards
4. Add to git and create a pull request
5. Get review from technical lead

## Content Guidelines

- **Clarity** - Write for developers of all experience levels
- **Accuracy** - Verify technical information
- **Examples** - Include code examples where relevant
- **References** - Link to authoritative sources
- **Maintenance** - Update outdated content regularly
